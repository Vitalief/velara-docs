# Velara - Epic Breakdown

> **Resequenced 2026-06-05** — see [Sprint Change Proposal](../sprint-change-proposal-2026-06-05.md). Epic order changed from infrastructure-first to feature-first (Registry & Execution prioritized; AWS provisioning relocated to Epic 7) because the client's AWS account and GitHub repos are not yet provisioned.
>
> **Updated 2026-06-08** — see [Sprint Change Proposal](../sprint-change-proposal-2026-06-08.md). Client design changes + compliance: Epic 9 → *Audit Log, Usage & Analytics* (adds Stories 9.4/9.5); Story 1.5 (per-page tab title) added; Epics 4/6/7 gain ACs; HIPAA + 21 CFR Part 11 named as compliance frameworks.
>
> **Added 2026-06-18** — see [Sprint Change Proposal](../sprint-change-proposal-2026-06-18.md). Client delivered the first real foundation skill (`velara-protocol-extractor`), a code-driven hybrid the current hybrid runtime cannot host. **New Epic 5.5** (positioned right after Epic 5) widens the hybrid runtime for code-driven trusted skills + adds a schema-versioned canonical output contract and `blocked` QA job state. Epics 2 & 3 stay `done`; Phase-1 isolation = per-skill venv (container isolation deferred to Epic 7). Headline gate: the client skill runs end-to-end on the platform. _(Renumbered 2026-06-25: created as Epic 10, moved to 5.5.)_
