5. **Compliance goal genuinely met**, not nominally.

## What Was Hard (cross-story patterns)

1. **Timezone / UTC anchoring — a class of bug across 4/5 stories.** 9.1 (partition bounds via session TZ GUC), 9.2 (bare-int date coercion + inverted range → both now 422), 9.3 (date presets local-TZ off-by-one), 9.4 (`date_trunc('week')` on session TZ). Everything touching dates/weeks/partitions needed *explicit* UTC.
2. **`hierarchy_path="org"` org-fence bug — the headline incident.** A user's real invocation never appeared in the Audit Log. `list_entries` fenced by `hierarchy_path <@ 'org_org_vitalief'`, but org-global rows carry `hierarchy_path="org"` and `'org' <@ 'org_org_vitalief'` = FALSE — silently hiding *every* org-global row for *every* caller. The 9.2 review saw it narrowly (admin events) and deferred as write-path; prod proved it a broad query-side org-isolation defect. Fixed via migration 0020 (real `org_id` column, backfilled, all 10 writers updated, regression test).
3. **`event_metadata` reserved-name trap rippled through all 4 layers** (ORM attr → Pydantic `validation_alias` → JSON `[object Object]` → JSONB extraction).
4. **Test/env friction every backend story** — `AUTH_BACKEND=dev` required, image must be rebuilt before pytest (source baked), recurring `test_ingest.py` MinIO false-positives.
5. **Live-service verification mishap (9.3)** — a dev restarted the user's *running* container with `AUTH_BACKEND=dev` and broke their active Cognito session.
