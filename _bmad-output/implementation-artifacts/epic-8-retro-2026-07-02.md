## What Went Well

1. **One ADR spined the whole epic.** The 8.1 "internal roles are org-global operators" ruling — internal staff run engagements, clients see only their granted slice, the bypass keys on `role` not `org` — governed 8.2's `RejectClient` guard, 8.3's positive `isInternal()` check, 8.4's run path, and 8.7's role tiering. One architect decision, four consistent downstream consumers. Intent → artifact.

2. **IP-safe schema discipline never cracked across four client surfaces.** Every client schema (`ClientJobRead`, `ClientEngagement*`, `ClientStudyRead`/`ClientLocationRead`, `ClientFanOutChild`) is an independent BaseModel with zero inheritance from internal schemas — the structural reason no skill internals reached a client token. The one place this discipline *lapsed* (8.6's `AttachedSkillRead.id: str` vs `Skill.id: UUID`) 500'd the entire attach UI and was caught in review — proving the rule matters.

3. **The 3-layer adversarial review caught real shipping bugs, repeatedly.** Three that single-layer review would have missed: the 8.2 blocked-job output leak (HIGH — a client could presign QA-held output), the 8.6 `AttachedSkillRead.id` type 500, and the 8.7 `GrantRole` Literal missing `admin` (which would have failed AC5's own rejection test). The review is the control that holds.

4. **Template-mirror discipline (8.6 ← 8.1) worked.** The `skill_attachment` model + migration + service was built by mirroring the `UserAccessGrant` grant pattern 1:1, as its ADR mandated — the attachment work reduced to "apply the grant shape to skills," and deviations (FK type = `postgresql.UUID`, not `String`) were caught early via type mismatch.

5. **Correct-course inserts were absorbed cleanly.** When 8.5/8.6/8.7 landed mid-sprint, each got its own ADR (where architectural) and its own adversarial review. 0 regressions, no completed story reopened. (The *planning* miss that created them is the key lesson below — but the *recovery* was disciplined.)

## What We Learned (Key Insights)

1. **The extra access-control scope was a PLANNING MISS, not a bonus — this is the retro's headline.** Original Epic 8 (8.1–8.4) answered "can a client *safely use* the portal?" It never scoped the *administrative* counterpart the grant model implies: a UI to manage grants (8.5), a skill-attachment model (8.6), and an internal-role tier to gate them (8.7). All three were inserted mid-sprint. **The team delivered them well — but delivery is recovery, and the goal is not needing to recover.** The tell was in 8.1's own ADR: it defined `POST /access-grants` and rejected internal-role grantees, which *implies* an admin surface and an authority tier. Enforce → administer → tier is a foreseeable dependency chain, and it was missed at planning. (Project Lead's own assessment: *"Should not have happened at all… a miss and wasted a lot of our time; the planning needs to be better."*)

2. **Cost of the miss was real: resequencing, redesign, and a review-born story.** 8.6 had to be resequenced *before* 8.4 (8.4's discovery depended on 8.6's attachment-backed query). 8.5 took a mid-story UX redesign (raw-UUID inputs → name pickers + cascading selectors) because the original spec's admin UX was unusable by humans. 8.7 was born from the 8.6 code review itself (reviewer flagged the flat two-role model). Clean outcomes, wasted cycles.

3. **A "sign-off" gate can silently degrade into a self-signed waiver under absorption pressure.** 8.4's D1 non-LD invocation widening (a client-authorization boundary change) required "architect sign-off recorded before merge." It was **self-asserted by the dev agent citing story guidance — not a real Winston ADR.** It merged under an explicit waiver. This is the single open governance thread from the epic. When a gate names an authority, the gate must produce *that authority's artifact*, not a proxy for it.

4. **When gates narrow, dead branches become reachable — and that's a coverage win to claim.** Pre-8.7, `_INTERNAL_ROLES == _GRANTOR_ROLES`, so the `_require_grantor` 403 branch was unreachable. 8.7's consultant demotion made it live, and the story added HTTP-level tests for it (including `test_consultant_can_still_invoke_after_demotion` to prove the run path stayed *un*gated). Narrowing a gate should always trigger "what branch just became reachable, and is it tested?"

5. **Test-infrastructure brittleness accrues quietly and taxes every later story.** Recurring across the epic: the `AUTH_BACKEND=cognito` container mismatch (dev HS256 tokens rejected — every integration run needs `-e AUTH_BACKEND=dev`); the 3 `test_client_surface.py` seed-FK failures that persisted across 8.3/8.4/8.6 gates on a *fresh* DB (a real seed defect, not stale state); the 8.1 `_postgres_reachable()` silent-skip that can drop the entire scope-filter suite in a misconfigured CI; and the "image bakes source — rebuild before pytest" tax. None are individually blocking; together they erode confidence and hide defects (the 8.7 `GrantRole` miss slipped partly because Postgres-gated integration tests didn't run in review).

## Decisions Made This Retro

- **D1 — The mid-epic access-control scope is recorded as a planning miss, not a win.** The team's recovery was clean; the root cause was incomplete epic planning (enforce → administer → tier chain not traced). Owner of the fix: John (PM).
- **D2 — Guardrail is a story-creation checklist item, not an epic-level pass.** Every `create-story` asks: *"Does this introduce new state, a boundary, or an enforcement point that needs a management UI, a reader/consumer, or a role to operate it — and is each covered by a story?"* Applied starting with Epic 9. (Chosen over a heavier epic-planning pass for being at the exact moment the gap is born.)
- **D3 — The 8.4 D1 waiver is PARKED, not dropped.** Project Lead decision: it does not block Epic 9; revisit later only if needed. Recorded explicitly in `deferred-work.md` (Epic 8 retro section) with both resolution paths (real ADR, or revert to LD-only) so future-us isn't surprised.
- **D4 — Both the planning-miss lesson and the parked D1 are written to `deferred-work.md`**, not left to live only in this retro doc — so they're discoverable at the point of Epic 9 work.

A missed consumer or reader in that chain compounds exactly the way the missed admin surface did in Epic 8. **D2's story-creation checklist is aimed precisely here.**
